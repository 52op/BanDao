# 办到 (BanDao) - 开发指南

## 项目简介

混合架构办公工具站：Next.js 前端 + Go 后端。纯前端工具在浏览器处理，CPU 密集型任务（PDF 转 DOC、图片压缩等）由 Go 服务处理。

## 技术栈

| 类别 | 技术 |
|------|------|
| 前端 | Next.js 16 (App Router) + React 19 + TypeScript |
| 样式 | Tailwind CSS 4 + Shadcn/ui |
| 后端 | Go + Gin + SQLite (modernc.org/sqlite) |
| 认证 | GoAuth SSO (RS256 JWT) + httpOnly session |
| 部署 | VPS + Nginx (bandao.it0731.cn) |

## 架构

```
Nginx (bandao.it0731.cn)
├── / → Next.js (端口 3000)
├── /admin/* → Next.js 后台管理
├── /api/auth/* → Next.js SSO 认证
├── /api/admin/* → Next.js 代理 → Go (X-Internal-Secret)
├── /api/convert/* → Next.js 代理 → Go 文件处理
└── /api/* → Go 公开 API (端口 8080)
```

**核心原则：**
- Go 是数据库唯一所有者，Next.js 不直接访问 SQLite
- Go 不验证 JWT，只信任 Next.js 的 X-Internal-Secret
- 公开数据（工具/分类/设置）浏览器直接访问 Go API
- 管理接口走 Next.js session 代理

## 快速开始

### 前端 (Next.js)

```bash
npm install
cp .env.example .env.local   # 配置环境变量
npm run dev                    # http://localhost:3000
```

### 后端 (Go)

```bash
cd BanDao-Backend
go build ./cmd/bandao
./bandao                      # 默认端口 8080
```

首次运行自动创建数据库和种子数据。

## 新增工具

### 纯前端工具

1. `src/tools/my-tool/index.tsx` — 创建组件
2. `src/tools/components.ts` — 添加 slug → lazy import
3. `src/lib/icon-map.ts` — 添加图标映射（如需要）

### 服务端工具

1. Go `internal/handlers/` 添加 handler
2. Go `cmd/bandao/main.go` 注册路由
3. Next.js `src/app/api/convert/` 添加代理路由
4. 前端组件调用 Next.js API

## 环境变量

```bash
# SSO 认证
AUTH_MODE=sso
NEXT_PUBLIC_GOAUTH_URL=https://auth.it0731.cn
GOAUTH_ISSUER=goauth
GOAUTH_PUBLIC_KEY=<RSA 公钥 PEM>

# Go 后端
BACKEND_URL=http://127.0.0.1:8080
INTERNAL_SECRET=<与 Go 共享的密钥>
```

## 后台管理

访问 `/admin` 需要 admin 角色。管理功能：
- Dashboard — 使用统计概览
- 工具管理 — 启用/禁用、需解锁标记
- 分类管理 — CRUD
- 站点设置 — 键值对编辑
- 用户管理 — 角色切换、解锁状态
- 解锁码 — 批量生成、复制

## 数据库 (Go 管理)

SQLite 位于 `BanDao-Backend/data/bandao.db`，表结构：
- `categories` — 工具分类
- `tools` — 工具元数据 (processor: frontend/go/external)
- `site_settings` — 站点设置
- `users` — SSO 影子账户
- `unlock_codes` — 解锁码
- `usage_stats` — 使用统计
