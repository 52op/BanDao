# 办到 (BanDao) - 开发指南

## 项目简介

纯前端办公工具站，基于 Next.js 14 + TypeScript + Tailwind CSS + Shadcn/ui。所有处理在浏览器端完成，无需后端。

## 技术栈

| 类别 | 技术 |
|------|------|
| 框架 | Next.js 14 (App Router) |
| 语言 | TypeScript |
| 样式 | Tailwind CSS 4 + Shadcn/ui |
| 图标 | Lucide React |
| PDF | pdf-lib / pdfjs-dist |
| 图片压缩 | browser-image-compression |
| 打包下载 | JSZip |

## 快速开始

```bash
npm install
npm run dev     # 开发服务器 http://localhost:3000
npm run build   # 生产构建
```

## 新增工具（3 步）

### 第 1 步：创建工具文件夹

在 `src/tools/` 下新建文件夹，包含 `index.tsx`：

```
src/tools/my-tool/
└── index.tsx
```

组件导出规则：
- **必须** `export default` 导出组件
- 组件内部为 `"use client"` 客户端组件
- 参考现有工具结构，按需引入通用组件

### 第 2 步：注册工具

编辑 `src/tools/registry.ts`，在 `tools` 数组中添加一条：

```ts
{
  slug: "my-tool",               // URL 路径标识
  name: "我的工具",               // 显示名称
  description: "一句话描述功能",   // 首页卡片描述
  icon: YourIcon,                 // Lucide 图标
  category: "image",              // 分类：pdf | image | text | privacy
  component: lazy(() => import("./my-tool")),
  needsUnlock: false,             // 大文件功能是否需要解锁
}
```

### 第 3 步：完成

路由 `/tools/my-tool` 自动生效，无需改框架代码。

## 批量处理模式

需要支持多文件的工具使用以下模式：

```tsx
import { BatchFileUpload, type BatchFileItem } from "@/components/tool/batch-file-upload";
import JSZip from "jszip";
import { downloadBlob } from "@/lib/download";
```

**状态管理：**
```tsx
const [files, setFiles] = useState<BatchFileItem[]>([]);
const [results, setResults] = useState<{ name: string; blob: Blob }[]>([]);
```

**处理流程：**
1. 用户上传 → `BatchFileUpload` 展示文件列表
2. 逐个处理 → 更新 `status: "processing" | "done" | "error"`
3. 下载 → 单文件直接下载，多文件用 `JSZip` 打包

## 工具分类

| 分类 | slug 前缀 | 说明 |
|------|-----------|------|
| `document` | pdf-*, markdown-* | 文档处理 |
| `image` | image-*, svg-*, watermark | 图片处理 |
| `developer` | text-*, json-* | 开发工具 |
| `utility` | qr-* | 实用工具 |

## 解锁机制

`needsUnlock: true` 的工具在处理 >5MB 文件时弹出公众号解锁弹窗。状态通过 `localStorage` 持久化。

相关组件：
- `src/components/unlock/unlock-context.tsx` — 全局解锁状态
- `src/components/unlock/unlock-dialog.tsx` — 弹窗 UI

## 通用组件

| 组件 | 用途 |
|------|------|
| `FileUpload` | 单文件上传（PDF、SVG 等） |
| `BatchFileUpload` | 批量文件上传（图片等） |
| `tool-page.tsx` | 工具页通用壳（标题+描述+内容区） |
| `downloadBlob()` | 触发浏览器下载 |

## 注意事项

- 所有处理纯前端，文件不上传服务器
- 图片处理使用 Canvas API，注意内存释放
- `URL.createObjectURL` 创建的 URL 用完需 `URL.revokeObjectURL` 释放
- 大文件处理建议用 `useWebWorker` 避免阻塞主线程
- 截图请保存至 `.playwright-mcp/snapshot/`，不要放项目根目录
