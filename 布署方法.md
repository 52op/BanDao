# 办到 (BanDao) 部署指南

## 架构概览

```
┌─────────────────────────────────────────────────────┐
│  方案 A: VPS 全栈部署 (推荐生产环境)                  │
│                                                      │
│  Nginx (bandao.it0731.cn)                           │
│  ├── / → Next.js (端口 3000)                        │
│  ├── /api/* → Go (端口 8080, 公开)                   │
│  ├── /api/auth/* → Next.js (SSO)                    │
│  └── /api/admin/* → Next.js → Go (内部认证)          │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│  方案 B: Cloudflare Pages + VPS 混合部署              │
│                                                      │
│  Cloudflare Pages (bandao.it0731.cn)                │
│  ├── / → Next.js 静态页面                           │
│  ├── /api/auth/* → Pages Functions (SSO)            │
│  └── /api/admin/* → Pages Functions → Go            │
│                                                      │
│  VPS (api.bandao.it0731.cn)                         │
│  └── /api/* → Go (端口 8080, 公开 + 内部)            │
└─────────────────────────────────────────────────────┘
```

## 方案 A: VPS 全栈部署

适合：有自有服务器、需要完整文件处理能力（PDF 转 DOC、图片压缩等）。

### 1. 服务器要求

| 项目 | 最低要求 |
|------|----------|
| 操作系统 | Ubuntu 22.04+ / Debian 12+ |
| CPU | 2 核 |
| 内存 | 2 GB |
| 磁盘 | 20 GB |
| Node.js | 18+ |
| Go | 1.21+ |

### 2. 安装系统依赖

```bash
# Node.js 18 LTS
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# Go 1.21+
wget https://go.dev/dl/go1.21.13.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.21.13.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
source ~/.bashrc

# LibreOffice (PDF 转 DOC)
sudo apt-get install -y libreoffice

# PM2 (进程管理)
sudo npm install -g pm2
```

### 3. 部署 Go 后端

```bash
# 创建目录
sudo mkdir -p /var/lib/bandao/{data,convert,log}
sudo chown -R $USER:$USER /var/lib/bandao

# 上传或克隆 BanDao-Backend
cd /opt
git clone <your-repo> BanDao-Backend
cd BanDao-Backend

# 构建
go build -o bandao-backend ./cmd/bandao

# 配置
cp config.toml config.toml.bak
cat > config.toml << 'EOF'
[server]
address = ":8080"

[database]
path = "/var/lib/bandao/data/bandao.db"

[security]
internal_secret = "你的随机密钥-越长越好"
allowed_origins = ["https://bandao.it0731.cn"]

[convert]
libreoffice_path = "/usr/bin/libreoffice"
temp_dir = "/var/lib/bandao/convert"
max_concurrent = 3
cleanup_age_hours = 24

[log]
dir = "/var/lib/bandao/log"
level = "info"
EOF

# systemd 服务
sudo cat > /etc/systemd/system/bandao-backend.service << 'EOF'
[Unit]
Description=BanDao Backend Service
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/BanDao-Backend
ExecStart=/opt/BanDao-Backend/bandao-backend
Restart=always
RestartSec=5
Environment=CONFIG_PATH=/opt/BanDao-Backend/config.toml

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable bandao-backend
sudo systemctl start bandao-backend

# 验证
curl http://localhost:8080/api/tools
```

### 4. 部署 Next.js 前端

```bash
cd /opt
git clone <your-repo> BanDao
cd BanDao

# 配置环境变量
cat > .env.local << 'EOF'
AUTH_MODE=sso
NEXT_PUBLIC_GOAUTH_URL=https://auth.it0731.cn
GOAUTH_ISSUER=goauth
GOAUTH_PUBLIC_KEY=<RSA 公钥 PEM>

BACKEND_URL=http://127.0.0.1:8080
INTERNAL_SECRET=<与 Go 配置相同的密钥>
SESSION_SECRET=<随机字符串用于 session 签名>
EOF

# 构建
npm install
npm run build

# PM2 启动
pm2 start npm --name bandao -- start
pm2 save
pm2 startup  # 开机自启
```

### 5. Nginx 配置

```nginx
# /etc/nginx/sites-available/bandao

server {
    listen 443 ssl http2;
    server_name bandao.it0731.cn;

    ssl_certificate     /etc/letsencrypt/live/bandao.it0731.cn/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/bandao.it0731.cn/privkey.pem;

    # Next.js 前端 + 页面
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Next.js 认证接口
    location /api/auth/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # Next.js 管理接口代理
    location /api/admin/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 文件转换接口（大文件上传）
    location /api/convert/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        client_max_body_size 50m;
    }

    # Go 后端公开 API（浏览器直接访问）
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        client_max_body_size 50m;
    }
}

server {
    listen 80;
    server_name bandao.it0731.cn;
    return 301 https://$host$request_uri;
}
```

```bash
sudo ln -s /etc/nginx/sites-available/bandao /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 方案 B: Cloudflare Pages + VPS 混合部署

适合：前端需要全球 CDN 加速、后端需要处理文件的场景。前端部署到 Cloudflare Pages，Go 后端保留在 VPS。

### 1. 前端部署到 Cloudflare Pages

#### 前置条件

- Cloudflare 账号
- 域名 `it0731.cn` 已接入 Cloudflare
- GoAuth 已注册 `bandao.it0731.cn` 客户端

#### 部署方式一：Git 连接（推荐）

1. 将代码推送到 GitHub/GitLab
2. Cloudflare Dashboard → Pages → Create a project
3. 连接仓库，设置：
   - **Framework preset**: Next.js
   - **Build command**: `npm run build`
   - **Build output directory**: `.next`
   - **Node.js version**: 18

4. 配置环境变量（Settings → Environment variables）：

```
AUTH_MODE=sso
NEXT_PUBLIC_GOAUTH_URL=https://auth.it0731.cn
GOAUTH_ISSUER=goauth
GOAUTH_PUBLIC_KEY=<RSA 公钥 PEM>
BACKEND_URL=https://api.bandao.it0731.cn
INTERNAL_SECRET=<与 Go 后端相同的密钥>
SESSION_SECRET=<随机字符串>
```

5. 触发部署

#### 部署方式二：Wrangler CLI

```bash
# 安装 Wrangler
npm install -g wrangler

# 登录
wrangler login

# 构建
npm run build

# 部署
wrangler pages deploy .next --project-name=bandao
```

#### 域名绑定

1. Cloudflare Dashboard → Pages → bandao → Custom domains
2. 添加 `bandao.it0731.cn`
3. 自动配置 DNS 和 SSL

### 2. Go 后端部署到 VPS

与方案 A 相同，但配置有调整：

```toml
[security]
internal_secret = "与 Cloudflare Pages 环境变量相同"
allowed_origins = [
    "https://bandao.it0731.cn",
    "https://bandao.pages.dev"  # Cloudflare Pages 默认域名
]
```

Nginx 配置简化（不需要代理 Next.js）：

```nginx
server {
    listen 443 ssl http2;
    server_name api.bandao.it0731.cn;

    ssl_certificate     /etc/letsencrypt/live/api.bandao.it0731.cn/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/api.bandao.it0731.cn/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        client_max_body_size 50m;
    }
}

server {
    listen 80;
    server_name api.bandao.it0731.cn;
    return 301 https://$host$request_uri;
}
```

### 3. DNS 配置

| 类型 | 名称 | 值 | 代理状态 |
|------|------|-----|----------|
| A | bandao | VPS IP | 已代理 (橙云) |
| A | api | VPS IP | 仅 DNS (灰云) |

> **注意**: `api` 记录不要开启代理，否则 Go 后端的文件上传/下载会受影响。

### 4. Cloudflare Pages Functions 处理

Cloudflare Pages 的 Functions 可以代理请求到 Go 后端。在 `src/app/api/` 下的 route.ts 文件已经通过 `BACKEND_URL` 环境变量指向 Go 后端，无需额外配置。

对于需要大文件上传的接口（如 PDF 转 DOC），Cloudflare 有以下限制：
- 免费版：请求体最大 100MB
- Workers 限制：CPU 时间 10ms（免费）/ 30s（付费）

如果文件处理是核心功能，建议使用方案 A（VPS 全栈）。

## 方案对比

| 特性 | 方案 A: VPS 全栈 | 方案 B: Cloudflare + VPS |
|------|-----------------|-------------------------|
| 前端 CDN | 无（需自建） | Cloudflare 全球 CDN |
| 文件处理 | 完整支持 | 受 Cloudflare 限制 |
| 部署复杂度 | 中等 | 较高（两套系统） |
| 成本 | VPS 费用 | VPS + Cloudflare（免费额度够用） |
| DDoS 防护 | 需自建 | Cloudflare 自带 |
| 适用场景 | 企业内部/功能完整 | 面向公网/需要加速 |

**推荐**：
- 个人项目/小团队 → 方案 A（简单直接）
- 面向公网/需要全球加速 → 方案 B
- 核心功能是文件处理 → 方案 A（避免 Cloudflare 限制）

## 常见问题

### Q: Go 后端启动报 "database is locked"

SQLite 在高并发写入时可能报错。确保 `config.toml` 中数据库路径使用 WAL 模式（已默认启用）：

```toml
[database]
path = "/var/lib/bandao/data/bandao.db?_journal_mode=WAL"
```

### Q: Cloudflare Pages 部署后 API 404

检查环境变量 `BACKEND_URL` 是否正确指向 Go 后端地址。Pages Functions 通过此变量代理请求。

### Q: PDF 转 DOC 失败

确保服务器已安装 LibreOffice：

```bash
which libreoffice
libreoffice --version
```

### Q: SSO 登录后跳转失败

1. 检查 GoAuth 是否注册了 `bandao.it0731.cn` 客户端
2. 检查 `GOAUTH_PUBLIC_KEY` 是否配置正确
3. 检查回调地址是否为 `https://bandao.it0731.cn/api/auth/sso/callback`
